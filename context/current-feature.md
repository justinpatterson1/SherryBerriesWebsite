# Current Feature: Forgot Password (Reset via Resend email)

## Status
In Progress

## Goals
- Add a "Forgot password?" affordance on the login page that lets a user request a reset link by email.
- Generate a one-time, time-limited password reset token server-side and persist it (mirrors the existing email-verification token flow).
- Send the reset link to the user's email via Resend (reuse `src/lib/email/resend.ts`), with a dev console fallback for the URL.
- Provide a reset page (e.g. `/reset-password?token=…`) where the user enters and confirms a new password.
- On valid token: hash the new password (bcrypt), update `User.password`, consume/delete the token; render success/expired/invalid states.
- Be enumeration-safe — requesting a reset for an unknown/unregistered email returns the same response as a known one.

## Notes
- **Entry point:** the login page at [src/app/login/page.tsx](../src/app/login/page.tsx) (credentials-only auth; social row already removed).
- **Resend wiring already exists** from the email-verification feature: [src/lib/email/resend.ts](../src/lib/email/resend.ts) (`sendVerificationEmail`, `EMAIL_FROM` default `SherryBerries <onboarding@resend.dev>`), and the token pattern in [src/lib/auth/verification.ts](../src/lib/auth/verification.ts) (32-byte hex, TTL, old tokens cleared). Reuse/mirror these for a `sendPasswordResetEmail` + reset-token helper.
- **Token storage:** prefer reusing the existing `VerificationToken` model (identifier = email) with a distinct prefix/namespace, OR confirm whether the schema needs a dedicated reset token. Verify against [prisma/schema.prisma](../prisma/schema.prisma) before deciding (avoid a migration if `VerificationToken` suffices).
- **Resend test-sender caveat (carried over):** the Resend test sender only delivers to the account owner until a domain is verified — dev path is the console-logged URL.
- **Auth:** password hashing uses `bcrypt` (not bcryptjs) per the existing codebase; `User.password` is the hashed field set at register time.
- Likely new files: `POST /api/auth/forgot-password` (request link), `POST /api/auth/reset-password` (consume token + set password), `/reset-password/page.tsx`, and a `src/lib/auth/password-reset.ts` helper.
